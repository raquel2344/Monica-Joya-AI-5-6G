# Module 05 - Open RAN and AI-Driven Networks

**Course:** ITAI 4370 - AI in 5G and 6G Communications and ORAN Networks  
**Student:** Monica Joya  |  **Term:** Summer 2026

## Overview

Module 05 covered Open RAN in depth, the shift from locked, single-vendor radio networks to open ones that different vendors can build together and that AI can steer. This was a reading and study module, so the work was going through the lecture and papers and pulling out how the architecture fits together. It set up the concepts used in the Module 03 assignment.

## Focus of this module

How an Open RAN is split into named parts and how AI plugs into it. The Radio Unit handles the antenna signal, the Distributed Unit handles real-time baseband, the Centralized Unit handles non-real-time baseband, the RAN Intelligent Controller (RIC) handles optimization, and the Service Management and Orchestration layer coordinates them.

## Readings that informed this module

- From DeepSense to Open RAN (survey)
- Advancements in Mobile Edge Computing (survey)

## What I learned

I learned that what makes it open is the standard interfaces between those parts, links named A1, E2, F1, E1, and O1, which break vendor lock-in and let an operator mix equipment from different vendors while AI reaches across the layers. That AI runs through the RIC and covers most of the stack: radio resource management, spectrum, energy efficiency, traffic prediction, and anomaly detection, while the Self-Organizing Network features handle configuration, optimization, healing, and protection.
